Claude Code workflow

Data sync

The canonical source of truth is the repo. Work with Claude Code to edit data/*.json files, then validate and push them to Supabase here. Pull in the other direction to generate a Claude Code prompt.

After editing with Claude Code — two buttons, done

  1. 01
    Come here after Claude Code commits always start here

    Whenever Claude Code has edited or updated any data/*.json file and pushed, open this page. The status tiles at § 01 show the last Supabase update time for each table — a quick indicator of how stale the database is.

  2. 02
    Click "Validate alignment" § 01 below

    This compares each repo JSON file against the live Supabase row. You'll get a row-by-row result. If everything shows IN SYNC you're done — nothing to push. If anything shows OUT OF SYNC, continue to step 3.

  3. 03
    Click "Push repo JSON to Supabase" § 02 below

    One click overwrites all four Supabase tables (profile, criteria, finances, contacts) with the current repo file versions. Watch the log — each table reports ✓ updated or an error. Takes a few seconds.

  4. 04
    Re-run Validate alignment to confirm optional but fast

    Click Validate alignment again. All rows should now show ✓ in sync and the badge should read IN SYNC. Done.

Reverse direction — if you edited data inside the app (shortlist, journey checks, map zones) and want to bring those values into the repo: use Pull from Supabase in § 02. It reads the current Supabase rows and generates a Claude Code prompt you can paste directly — Claude Code will write the values back to the relevant data/*.json files and commit.
What syncs: 15 user-state tables live in Supabase as source of truth (see status bar below). Content mirrors — areas (195 rows) and house_types (15 rows) — are repo JSON pushed to Supabase via tools/backfill-content-direct.mjs. debts_credit_cards, debts_student_loans, debts_other have no app UI yet; rows will appear here when data is entered.
